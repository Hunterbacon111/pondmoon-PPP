## 第三步：搭建你的 AI 助手 —— OpenClaw

这是整个套餐里最"工程"的一步，但用 Claude Code 帮你装，会简单很多。

### 3.1 养金鱼比喻（核心概念）

```
养 AI 助手 = 养金鱼

🐟 金鱼     = OpenClaw（你的 AI 助手）
🏠 鱼缸     = 服务器（24 小时运行的机器）
🔲 保护区   = Docker（鱼缸里的隔离罩）
🧠 鱼粮     = 大模型 API（Claude API 或 Kimi API）
```

### 3.2 买个鱼缸（选服务器）

你需要一台 **24 小时运行** 的服务器。推荐方案：

| 方案 | 价格 | 特点 |
|------|------|------|
| **DigitalOcean**（推荐） | ~$10-12/月 | 服务器在纽约，稳定好用 |
| 阿里云（海外节点） | 更便宜 | 国产服务商，海外节点可选 |
| 腾讯云（海外节点） | 更便宜 | 同上 |

> **配置建议**：内存选 **2GB 以上**。因为 AI 助手未来可能同时并行多个任务，内存太小会卡。

### 3.3 用 Claude Code 远程安装（重点）

**为什么要通过 Claude Code 来装，而不是自己装？**

安装 OpenClaw 涉及大量网关（gateway）和网络节点配置，自己弄容易踩坑。让 Claude Code 来装的好处：
- 出了问题它能实时帮你排查
- 它会自动处理各种配置细节
- 你只需要跟它对话就行

#### 操作流程：

```
第 1 步：获取服务器 SSH 连接信息
         ┌─────────────────────────────────────┐
         │  例如：ssh root@你的服务器IP           │
         └─────────────────────────────────────┘
                          │
                          ▼
第 2 步：在 Claude Code 中建立 SSH 连接
         （本地 Claude Code ←→ 远程服务器 打通）
                          │
                          ▼
第 3 步：告诉 Claude Code 在 Docker 里安装 OpenClaw
         ┌─────────────────────────────────────┐
         │  Docker = 保护区                      │
         │  即使 OpenClaw 出问题，也不会影响       │
         │  服务器上的其他文件和服务                │
         └─────────────────────────────────────┘
                          │
                          ▼
第 4 步：OpenClaw 官网提供一行代码完成下载
         （过程非常顺滑）
                          │
                          ▼
第 5 步：配置大模型 API
         ┌──────────────────┬──────────────────┐
         │  不差钱 → Claude API  │  省钱 → Kimi API   │
         └──────────────────┴──────────────────┘
                          │
                          ▼
第 6 步：让 Claude Code 帮你开网关（gateway）
         （自己弄会很花时间，交给 Claude Code）
```

### 3.4 在保护区里放鱼（为什么用 Docker）

> **比喻**：你的鱼（OpenClaw）是条斗鱼，非常敏感、容易过激。所以你在鱼缸（服务器）里又加了一个保护区（Docker），让斗鱼在里面安全地适应生活。如果它搞出什么问题，你在保护区里就能修改，**不会污染鱼缸里的其他生态**。

技术上说：Docker 是一个隔离的运行环境，OpenClaw 的所有文件和配置都在里面。即使配置出错，删掉重来就行，不影响服务器本身。

## FAQ

**Q：我完全不懂代码，能搞吗？**
A：能。Claude Code 本身就是帮你写代码的，你只需要用自然语言描述需求。安装服务器和 OpenClaw 的过程中，遇到问题都可以问 Claude Code。

**Q：为什么不直接用 ChatGPT / Claude 聊天，还要搞 OpenClaw？**
A：ChatGPT 和 Claude 是"被动"的——你问它才答。OpenClaw 是"主动"的——你可以给它布置定时任务、连接各种服务、让它自主执行工作，更像一个真正的助手。

**Q：Kimi 2.5 是什么？**
A：Kimi 是月之暗面（Moonshot AI）出品的大模型，性能不错，API 价格比 Claude 便宜很多。作为 OpenClaw 的"大脑"来用，性价比很高。

**Q：服务器挂了怎么办？**
A：因为用了 Docker，你的 OpenClaw 配置都在容器里。服务器重启后，Docker 容器会自动恢复。实在出问题，用 Claude Code 重新连上去修就行。
